# Technology Principles

## Architecture

Build once.

Reuse everywhere.

------------------------------------------------

Prefer platform modules over standalone scripts.

------------------------------------------------

Every dataset should have metadata.

------------------------------------------------

Every pipeline should be observable.

------------------------------------------------

Every recommendation should be explainable.

------------------------------------------------

Prefer historical intelligence over temporary signals.

------------------------------------------------

No duplicated logic.

------------------------------------------------

Every module must belong to one platform layer.

------------------------------------------------

Keep APIs stable.

------------------------------------------------

Documentation is part of the product.

------------------------------------------------

The platform should become stronger every day.
